# Changelog

_Public app releases._

## Version 1.1.0

### Changed
- Redesigned the **Caravan Add** flow with a cleaner, more structured form experience.
- Improved **route and date selection** in caravan creation for a more polished setup flow.
- Refreshed the **Passenger Add** screen with clearer sections and less friction when entering data.
- Made passenger creation more flexible by improving field organization and reducing required information where appropriate.

### Improved
- Added **smarter passenger deduplication**, helping prevent duplicate passenger records.
- Improved how passenger data is merged when existing and new information overlap, keeping better-quality details when possible.
- Increased reliability of **images and thumbnails**, especially in edge cases involving iCloud availability.
- Hardened startup and background maintenance behavior to improve overall app stability.
- Polished localized **permission prompt** handling and related app text infrastructure.

### Notes
- Included additional internal reliability, migration, and data integrity improvements to support a more stable experience.

## Version 1.0.0

### Added
- Global location support using Apple Maps
- Trip creation using map-based locations (departure and destination)
- Passenger management
- Passenger check-in flow
- Random passenger raffle tool
- Offline support for trip and passenger management
- Ctrlavana Pro subscription (7-day free trial)

### Changed
- Replaced the Brazil ZIP-code location system with map-based location selection
- Updated the trip creation flow to support global locations
- Updated the passenger management interface

### Notes
- Earlier versions existed as a personal project focused on Brazil
- Version 1.0.0 marks the first release under the company account
